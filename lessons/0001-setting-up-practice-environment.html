Setting Up Your CKAD Practice Environment

Lesson 001 · Estimated time: 20 minutes · Domain: Exam Preparation

The CKAD exam is 100% hands-on — you solve problems in a real terminal connected to real clusters. There's no multiple choice. You need a local cluster you can create, destroy, and recreate freely so you can practice daily without cost or friction.

What you'll set up

  • Docker Desktop — container runtime (you may already have this)
  • kind — spins up Kubernetes clusters inside Docker containers
  • kubectl — the CLI you'll use in the actual exam
  • kubectl aliases — speed shortcuts for the exam

Why kind over minikube?

kind (Kubernetes IN Docker) is lighter, faster to create/destroy, and supports multi-node clusters trivially. A fresh cluster spins up in ~30 seconds. You can have multiple clusters simultaneously for different practice scenarios. kind documentation.

Steps

Step 1 — Install prerequisites

If you don't have Docker Desktop yet, install it first:

brew install --cask docker

Open Docker Desktop and let it start. Then install kind and kubectl:

brew install kind kubectl
Step 2 — Create your first cluster
kind create cluster --name ckad-practice

This creates a single-node cluster. It takes about 30 seconds.

Step 3 — Verify it works
# Check kubectl is connected
kubectl cluster-info

# You should see something like:
# Kubernetes control plane is running at https://127.0.0.1:PORT

# List nodes
kubectl get nodes

# You should see one node with STATUS: Ready

If kubectl get nodes shows a node with status Ready, your environment is working. Everything from here on is optimization.

Step 4 — Set up exam-style aliases

The exam environment pre-configures some aliases. Set these up now so they become muscle memory:

# Add to your ~/.zshrc (or ~/.bashrc)
alias k='kubectl'
alias kn='kubectl config set-context --current --namespace'

# Autocomplete for the alias
source <(kubectl completion zsh)
complete -o default -F __start_kubectl k

# Apply changes
source ~/.zshrc

In the real exam, kubectl autocomplete and the alias k=kubectl are pre-configured. But you still need to be fast. Start using k instead of kubectl in all your practice from day one.

Step 5 — Test a quick workflow

Let's verify the full loop works — create something, inspect it, delete it:

# Create a pod imperatively (exam-critical skill)
k run nginx --image=nginx

# Check it's running
k get pods

# Look at the YAML it generated
k get pod nginx -o yaml

# Delete it
k delete pod nginx

If you successfully created and deleted a pod, your practice environment is fully operational. You're ready for lessons.

Cluster lifecycle

You'll use these commands regularly:

# Delete cluster when done for the day (optional — it's lightweight)
kind delete cluster --name ckad-practice

# Recreate fresh cluster anytime
kind create cluster --name ckad-practice

# List your clusters
kind get clusters

Being able to destroy and recreate your cluster in seconds is a superpower. Made a mess? Broke something? Don't debug it — nuke it and start fresh. This is how you should practice for the exam too: each question gets a clean slate.
